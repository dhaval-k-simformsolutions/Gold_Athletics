29. Pending Tasks and TODO

Are pending things being marked properly in code?

Not Following

Not Following points:

  • The codebase lacks consistent use of standard keywords like TODO or FIXME to mark pending tasks or incomplete implementations.
  • A search for these standard markers doesn't yield results, suggesting that either pending tasks aren't properly marked or they are marked using non-standard notation.
  • There is a commented code block in `lib/source/bottom_tab/main_campagin_notification/controller/save_fcm_token_controller.dart` that indicates incomplete implementation without proper TODO markers.

Recommendation:

  • Implement a consistent system for marking pending tasks using standard keywords (TODO, FIXME) that can be easily tracked by IDE tools.
  • Add clear descriptions and assignees to TODOs where applicable: `// TODO(username): Implement token refresh mechanism by [date]`
  • Consider using a linter rule to prevent TODOs from staying in the codebase for too long without resolution.
